										Data Security and Privacy Statement

**Overview**

This is a data security and privacy statement for the Jira marketplace plugin "Sprint Ops Center" by iSynergy TechSys Pvt. Ltd.

**App Architecture**

The plugin is developed using Atlassian's forge platform. The plugin works inside the Atlassian environment and no external server or database is required. All required raw data is fetched through the forge platform and then processed to provide insights. Any settings and configuration required for the app to work are saved with your Jira storage within the Atlassian environment.

**Privacy**

We do not collect any personal data from your use of our plugin. Additional information we receive is usage statistics and licensing information through Atlassian's marketplace API.  
For capacity planning, sprint demos, retrospectives, all the additional data is saved in the storage provided by Jira and your data never leaves the Jira environment.

**Cloud plugin permissions Scope**

Our plugin requires various read permissions to be able to read Jira tickets and derive insights. We also need write permission to write issue data entered through our apps to create tickets automatically from the retrospective action items and to use the Jira storage for settings and configuration.

**Data Storage/Data Retention**

All the required data are stored in the Jira storage and no personally identifiable information leaves the Jira storage.

For the plugin to work with all the features, the following fields are stored in Jira storage and does not leave the cloud instance. All the data is already present with the Jira account, the plugin needs a read only access to derive insights at the individual level:

1.  User ID
2.  Email
3.  Capacity per day as entered in our app
4.  Team Holidays as entered in our app
5.  Sprint ID
6.  Team Capacity per day as entered in our app
7.  Sprint demo date to track the demo readiness with the linked stories.
8.  Sprint retrospective comments and action items. Comments are not tied to an individual user.

All the above mentioned data is stored with the Jira cloud storage in the plugin itself. There is no external server or database.
